# RELEASE_CHECKLIST

## 1. 목적

이 문서는 Nightseed Survivor의 릴리즈 전 확인 항목을 정의한다.

최종 배포 판단은 로컬 환경보다 GitHub Actions 결과와 릴리즈 산출물 기준을 우선한다.

---

## 2. 릴리즈 전 확인

### 문서

- [ ] `CHANGELOG.md` 최신화
- [ ] `HISTORY.md` 최신화
- [ ] `docs/GAME_SPEC.md`와 실제 동작이 모순되지 않음
- [ ] `docs/BALANCE.md`와 주요 수치가 모순되지 않음
- [ ] `.agent/progress.md` 최신화
- [ ] `.agent/tasks.md` 후속 작업 정리
- [ ] `.agent/decisions.md` 주요 판단 기록

### 버전

- [ ] Godot 프로젝트 버전 확인
- [ ] 릴리즈 태그 버전 확인
- [ ] `CHANGELOG.md` 버전 확인
- [ ] GitHub Release 제목 확인

### 보안

- [ ] API 키, 토큰, 인증서, 릴리즈 키가 커밋되지 않음
- [ ] 민감 정보가 로그에 포함되지 않음
- [ ] 불필요한 네트워크 권한 없음
- [ ] 광고, 분석, 로그인 SDK 없음

### 라이선스

- [ ] 사용한 애셋의 라이선스 확인
- [ ] 저작권 불명확한 이미지/음원 없음
- [ ] 기존 상용 게임의 이름, UI, 아이콘, 아트 복제 없음

---

## 3. 빌드 확인

### 로컬

- [ ] Godot headless 실행

```bash
godot --headless --path godot --quit
```

- [ ] 핵심 플레이 플로우 확인
  - [ ] 이동
  - [ ] 적 스폰
  - [ ] 자동 공격
  - [ ] 경험치 획득
  - [ ] 레벨업
  - [ ] 게임오버
  - [ ] 클리어
  - [ ] 저장/로드

### CI

- [ ] GitHub Actions 성공
- [ ] 실패 시 로그 확인 후 수정
- [ ] 재실행 결과 확인

권장 명령:

```bash
gh run list --limit 10
gh run view <RUN_ID> --log-failed
```

---

## 4. Expected Assets 확인

릴리즈에는 실제 구동 가능한 산출물이 포함되어야 한다.

Expected Assets:

```text
APK
ZIP
```

확인 항목:

- [ ] APK 파일 존재
- [ ] ZIP 파일 존재
- [ ] 파일 크기가 0이 아님
- [ ] 예상 확장자와 파일명이 올바름
- [ ] 릴리즈 노트에 설치/실행 방법 포함
- [ ] 다운로드한 산출물이 정상적으로 열림

---

## 6. 금지 사항

- 검증하지 않은 빌드를 성공으로 기록하지 않는다.
- 릴리즈하지 않은 변경 사항을 릴리즈 완료처럼 쓰지 않는다.
- 파일명만 있고 실제 실행 파일이 없는 릴리즈를 완료로 보지 않는다.
- 태그 버전과 앱 내부 버전이 다르면 릴리즈하지 않는다.
