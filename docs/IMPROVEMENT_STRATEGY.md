# IMPROVEMENT_STRATEGY

## 1. 목적

이 문서는 기존 뱀서라이크 장르에서 자주 느껴지는 아쉬움을 개발 초기에 먼저 반영하기 위한 설계 가이드다.

이 내용은 특정 게임에 대한 통계적 분석이 아니라, Nightseed Survivor의 MVP 설계에 적용할 개선 가설이다.

목표는 단순히 기존 장르를 복제하는 것이 아니라, 작은 규모에서도 더 명확한 성장감, 덜 피곤한 화면, 반복 플레이 동기를 만드는 것이다.

---

## 2. 선반영할 문제와 대응 방향

| 기존 장르에서 느껴질 수 있는 아쉬움 | Nightseed Survivor 대응 방향 |
|---|---|
| 초반 1~2분이 지루함 | 30초 안에 첫 레벨업 발생 |
| 무기 선택지가 반복적으로 느껴짐 | 무기별 역할과 사용감을 명확히 분리 |
| 성장 체감이 약함 | 레벨 4~5에서 패턴 변화 제공 |
| 화면이 너무 혼잡함 | 이펙트 제한, 데미지 숫자 MVP 제외 |
| 적 종류가 많아도 차이가 작음 | 적마다 역할을 명확히 분리 |
| 모바일에서 조작 피로가 큼 | 공격 버튼 없이 이동만 조작 |
| 실패 후 재도전 동기가 약함 | 골드와 영구 강화로 누적 성장 제공 |
| 밸런스 수정이 어려움 | 수치를 문서/데이터 중심으로 분리 |
| 보스가 단순한 체력 덩어리로 느껴짐 | 보스 등장 시 웨이브 압박과 목표감을 동시에 제공 |

---

## 3. 초반 지루함 개선

### 문제

뱀서라이크 장르는 초반에 적이 적고 무기가 약하면 지루하게 느껴질 수 있다.

### 정책

- 첫 30초 안에 레벨업을 발생시킨다.
- 1분 안에 적 2종을 경험하게 한다.
- 초반 무기 발동 간격은 너무 길게 잡지 않는다.
- 시작 무기는 반드시 적을 자동 추적하거나 맞추기 쉬워야 한다.

### 구현 기준

```text
first_level_up_target_time = 30 seconds
starting_weapon_cooldown <= 1.2 seconds
early_enemy_spawn_interval <= 1.0 seconds
```

---

## 4. 성장 체감 개선

### 문제

레벨업이 단순히 피해량 +5% 같은 수치 증가만 반복되면 선택의 재미가 약해진다.

### 정책

- 낮은 레벨에서는 수치 증가를 제공한다.
- 높은 레벨에서는 패턴 변화나 체감 가능한 변화를 제공한다.
- 무기마다 레벨 4 또는 5에서 확실한 변화가 있어야 한다.

### 예시

| 무기 | 고레벨 변화 |
|---|---|
| Moon Dagger | 투사체 수 증가, 관통 추가 |
| Spirit Orb | 구체 수 증가, 회전 반경 증가 |
| Fire Wisp | 폭발 횟수 증가 |
| Thorn Ring | 넉백 증가, 쿨다운 감소 |
| Star Needle | 발사 수 증가, 관통 추가 |

---

## 5. 선택지 반복감 개선

### 문제

레벨업 선택지가 매번 비슷하면 사용자는 선택을 하지 않고 습관적으로 누르게 된다.

### 정책

- 선택지는 공격형, 방어형, 편의형이 섞이도록 한다.
- 새 무기와 기존 무기 강화가 적절히 섞이도록 한다.
- 플레이어가 이미 가진 빌드 방향을 강화할 수 있어야 한다.
- 완전히 무작위가 아니라 현재 상태를 고려한다.

### 선택지 생성 우선순위

```text
1. 보유 무기 강화 후보
2. 보유 패시브 강화 후보
3. 신규 무기 후보
4. 신규 패시브 후보
5. 회복 또는 골드 보상 후보
```

---

## 6. 화면 혼잡 개선

### 문제

적, 투사체, 이펙트, 데미지 숫자가 한꺼번에 많아지면 모바일에서 화면을 읽기 어렵다.

### 정책

- MVP에서는 데미지 숫자를 기본 비활성화한다.
- 이펙트는 짧고 작게 유지한다.
- 적과 플레이어의 실루엣을 명확히 구분한다.
- 경험치 보석은 작지만 색 대비를 강하게 둔다.
- 위험 영역은 붉은색 계열로 통일한다.

### 성능 기준

```text
max_enemies_on_screen = 150 to 250
max_projectiles_on_screen = 200
max_pickups_on_screen = 300
```

모바일 성능 이슈가 있으면 적 수를 먼저 줄이고, 그 다음 이펙트를 줄인다.

## 7. 적 역할 차별화

### 문제

적 종류가 많아도 모두 플레이어를 따라오기만 하면 실제 플레이 차이가 작다.

### 정책

| 적 | 역할 |
|---|---|
| Shadow Slime | 기본 압박 |
| Bone Bat | 빠른 접근으로 회피 요구 |
| Hollow Knight | 탱커로 공간 점유 |
| Rot Hound | 중반 이후 돌진 압박 |
| Ancient Shade | 마지막 생존 목표 |

---

## 8. 반복 플레이 동기

### 문제

죽었을 때 아무것도 남지 않으면 반복 플레이 의지가 떨어진다.

### 정책

- 실패해도 골드는 남는다.
- 골드는 영구 강화에 사용된다.
- 영구 강화는 너무 강하지 않되 다음 플레이가 쉬워지는 느낌을 준다.
- 최고 생존 시간과 총 처치 수를 기록한다.

---

## 9. 모바일 피로도 개선

### 정책

- 공격 버튼을 만들지 않는다.
- 이동 조작만으로 플레이 가능해야 한다.
- 레벨업 선택은 큰 카드 3개로 제공한다.
- UI는 엄지로 누르기 쉬운 크기를 유지한다.
- 장시간 손가락 고정 피로를 줄이기 위해 조이스틱 감도를 조정 가능하게 한다.

---

## 10. 구현 우선순위

1. 재미 검증 가능한 placeholder 루프
2. 첫 레벨업 타이밍 조정
3. 무기별 체감 차이 구현
4. 웨이브 압박 조정
5. 영구 강화 추가
6. 실제 그래픽 애셋 교체
7. 이펙트와 사운드 추가
