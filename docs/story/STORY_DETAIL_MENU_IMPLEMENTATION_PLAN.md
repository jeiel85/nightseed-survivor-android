# Story Detail Menu Implementation Plan

## 목적

인게임 스토리는 플레이 흐름을 방해하지 않는 핵심 요약 자막으로 유지하고, 메인 메뉴의 스토리 화면에서는 같은 사건을 더 자세한 장부/연대기 형태로 읽을 수 있게 분리한다.

현재 StoryUI는 `story_dialogues.json`의 인게임 자막을 다시 보여주는 구조라서, 스토리 메뉴가 "다시 보기" 이상으로 확장되기 어렵다. 다음 구현에서는 런타임 자막 데이터와 상세 읽기 데이터를 분리한다.

## 기준 문서

- `docs/story/STORY_FINAL_SPEC.md`
- `docs/story/STORY_NIGHTSEED_LORE.md`
- `docs/story/STORY_STAGE_DIALOGUE.md`
- `docs/story/STORY_UI_COPY.md`
- `.agent/decisions.md`의 런타임 스토리 연결 결정

## 핵심 방향

### 인게임 자막

- 시작, 보스 등장, 클리어처럼 중요한 진행 지점에서만 표시한다.
- 한 번에 1~2줄 정도의 짧은 요약만 사용한다.
- 기존 `StoryBanner` 비차단 자막 구조를 유지한다.
- 기존 `godot/data/story_dialogues.json`는 인게임 요약 자막 전용 데이터로 유지한다.

### 스토리 메뉴

- 스테이지별 상세 챕터를 제공한다.
- 인게임 자막을 상단 요약으로 재사용하되, 본문은 별도 상세 텍스트를 표시한다.
- 각 챕터는 Nightseed 정보 공개 순서를 따른다.
- 해금 전에는 스포일러를 숨기고, 해금 후에는 상세 내용을 읽을 수 있게 한다.
- 클리어 여부에 따라 추가 단서나 회고 문단을 여는 구조를 우선 검토한다.

## 데이터 구조 제안

새 파일:

```text
godot/data/story_chapters.json
```

예상 스키마:

```json
{
  "version": 1,
  "chapters": {
    "forest": {
      "title": {
        "ko": "메아리의 숲",
        "en": "Forest of Echoes"
      },
      "summary": {
        "ko": "숲은 방랑자가 잃어버린 이름의 흔적을 처음으로 일깨운다.",
        "en": "The forest first stirs the Vagrant's lost name."
      },
      "body": [
        {
          "unlock": "stage_unlocked",
          "heading": {
            "ko": "숲이 기억하는 이름",
            "en": "The Name the Forest Remembers"
          },
          "text": {
            "ko": "상세 본문...",
            "en": "Detailed body..."
          }
        },
        {
          "unlock": "stage_cleared",
          "heading": {
            "ko": "남은 조각",
            "en": "The Remaining Fragment"
          },
          "text": {
            "ko": "클리어 후 열리는 상세 단서...",
            "en": "A clue unlocked after clearing the stage..."
          }
        }
      ],
      "revealed_terms": ["nightseed"]
    }
  }
}
```

잠금 조건 후보:

- `stage_unlocked`: 스테이지가 해금되면 표시
- `stage_cleared`: 해당 스테이지 최초 클리어 후 표시
- `campaign_cleared`: 마지막 스테이지 클리어 후 표시

## 구현 순서

1. `.agent/tasks.md`에 `feat: 스토리 메뉴 상세 장부 데이터 분리` 작업을 등록한다.
2. `godot/data/story_chapters.json`를 추가한다.
3. `Story.gd`에 상세 챕터 로드와 현 언어 기준 조회 API를 추가한다.
   - `get_stage_chapter(stage_id: String) -> Dictionary`
   - `get_chapter_sections(stage_id: String) -> Array`
   - 잠금 조건 판정은 `Story.gd` 또는 `StoryUI.gd` 중 기존 책임에 맞춰 최소 구현한다.
4. `StoryUI.gd`를 수정한다.
   - 카드 상단에는 챕터 라벨, 스테이지명, 짧은 요약을 표시한다.
   - 본문에는 `story_chapters.json`의 상세 섹션을 표시한다.
   - 인게임 자막은 "요약 자막" 또는 "전투 중 기록" 정도의 접힌/짧은 섹션으로만 남긴다.
5. `Localization.gd`에 필요한 UI 키를 추가한다.
   - 예: `story_section_summary`, `story_section_detail`, `story_section_battle_quotes`, `story_locked_clear_required`
6. 문서를 갱신한다.
   - `docs/story/README.md`: 새 데이터 파일 목적 추가
   - `docs/ARCHITECTURE.md`: `story_chapters.json` 데이터 역할 추가
   - `HISTORY.md`, `CHANGELOG.md`, `.agent/progress.md`: 실제 구현 후 결과 기록
7. 검증한다.
   - JSON 문법 검사
   - `godot --headless --path godot --quit`
   - 가능하면 `StoryUI.tscn` 단독 로드
   - 해금/미해금/클리어 상태별 표시 확인

## 스토리 작성 가이드

- `STORY_STAGE_DIALOGUE.md`의 정보 공개 순서를 유지한다.
- 인게임 자막보다 길어도 되지만, 모바일 화면에서 한 카드가 지나치게 길어지지 않게 섹션을 나눈다.
- Nightseed는 악마, 숭배, 계약, 제물 의식과 연결하지 않는다.
- 각 스테이지는 "장소의 기억 왜곡"을 중심으로 설명한다.
- 보스는 단순 악당이 아니라 봉인과 기억에 묶인 파수꾼으로 다룬다.
- 한국어/영어 병기를 기본으로 한다.

## 스테이지별 상세 내용 방향

| 스테이지 | 상세 메뉴에서 드러낼 내용 |
|---|---|
| Forest of Echoes | Nightseed라는 이름이 처음 등장하고, 숲이 방랑자의 이름을 기억하고 있음을 암시 |
| Frozen Wastes | 밤의 씨앗이 잊혀진 맹세와 이어져 있음을 설명 |
| Twilight Sanctum | 방랑자가 봉인을 지킨 자이자 연 자라는 모순을 부각 |
| Inferno Chasm | Nightseed는 단순히 태우거나 파괴할 수 없다는 사실을 확장 |
| Cursed Tomb | 방랑자가 마지막 기사였고, 잃어버린 이름이 봉인과 연결되어 있음을 상세히 공개 |

## 상세 스토리 초안

이 섹션은 다음 세션에서 `godot/data/story_chapters.json`로 옮길 본문 초안이다. 문장은 모바일 StoryUI에서 읽기 쉽도록 짧은 단락 단위로 나누고, 각 스테이지는 `summary`, `stage_unlocked`, `stage_cleared` 섹션으로 구성한다.

### Forest of Echoes

Summary:

```text
숲은 방랑자가 잃어버린 이름의 흔적을 처음으로 일깨운다.
```

Stage unlocked — 숲이 기억하는 이름:

```text
메아리의 숲은 오래전부터 사람의 목소리를 따라 하는 곳으로 알려져 있었다. 그러나 방랑자가 그 밤 숲에 들어섰을 때, 나무들이 되풀이한 것은 누군가의 목소리가 아니었다. 그것은 이름이 있어야 할 자리에 남은 텅 빈 울림이었다.

잎사귀는 바람도 없이 떨렸고, 오래된 뿌리는 땅속에서 한 방향을 가리켰다. 숲은 길을 숨기지 않았다. 오히려 너무 오래 기다렸다는 듯, 달빛이 닿는 자리마다 희미한 길을 열어 주었다.

방랑자는 자신의 이름을 기억하지 못한다. 하지만 숲은 그가 무엇을 잃었는지 알고 있는 듯했다. 나무껍질에 남은 금빛 균열은 그의 검에 난 금과 닮아 있었고, 그 끝에는 방랑자가 처음 듣는 말이 남아 있었다.

밤의 씨앗.
```

Stage cleared — 남은 조각:

```text
숲의 파수꾼은 방랑자를 죽이려 했던 것이 아니었다. 그는 숲 아래 잠든 것을 지키고 있었다. 그것이 깨어난다면 숲만 무너지는 것이 아니라, 숲이 간직해 온 모든 이름이 한꺼번에 지워질 터였다.

전투가 끝난 뒤, 방랑자의 손끝에 작은 기억의 조각이 남았다. 그것은 얼굴도, 장소도 아니었다. 누군가가 검을 건네며 낮은 목소리로 말하던 순간이었다.

이름을 잃어도 길은 잃지 마라.
```

### Frozen Wastes

Summary:

```text
얼어붙은 황무지는 밤의 씨앗이 잊혀진 맹세와 이어져 있음을 드러낸다.
```

Stage unlocked — 멈춘 약속:

```text
결빙의 황무지에는 눈이 내리지 않는다. 이곳을 뒤덮은 흰 층은 하늘에서 떨어진 눈이 아니라, 오래전에 지켜지지 못한 약속들이 얼어붙어 생긴 흔적이었다.

방랑자가 발을 디딜 때마다 얼음 아래에서 흐릿한 문장들이 떠올랐다. 맹세의 한 조각, 이름의 한 조각, 그리고 누군가 끝내 지키지 못한 마지막 말. 그것들은 제대로 읽히기도 전에 다시 얼음 속으로 가라앉았다.

결빙의 수호자는 문 앞에서 방랑자를 막아섰다. 그가 요구한 것은 힘의 증명이 아니라 기억의 증명이었다. 무엇을 지키려 했는지 모르는 자는, 자신이 무엇을 잃었는지도 알 수 없기 때문이다.

밤의 씨앗은 어둠 속에서만 자라는 것이 아니었다. 잊혀진 맹세가 차갑게 굳어 버린 곳에서도, 그것은 조용히 뿌리를 내리고 있었다.
```

Stage cleared — 검 끝의 결:

```text
수호자가 쓰러진 자리에서 얼음이 갈라졌다. 그 안에는 보물도, 시체도 없었다. 대신 검 끝에 닿는 순간 빛으로 흩어지는 문장 하나가 남아 있었다.

나는 문을 닫겠다.

방랑자는 그 말이 자신의 것인지, 아니면 오래전 누군가에게 들었던 말인지 알 수 없었다. 다만 검의 균열 하나가 금빛으로 메워지며, 오래된 맹세의 흔적을 다시 밝혀 내기 시작했다.
```

### Twilight Sanctum

Summary:

```text
황혼의 성소는 방랑자가 봉인을 지킨 자이면서 동시에 연 자였다는 모순을 보여준다.
```

Stage unlocked — 둘 다 사실인 문:

```text
황혼의 성소에는 완전한 낮도, 완전한 밤도 존재하지 않는다. 모든 빛은 문턱에서 멈추고, 모든 그림자는 닫히기 직전의 틈에서 흔들린다.

성소의 벽에는 같은 인물이 두 번 새겨져 있었다. 한쪽의 그는 검으로 문을 닫고 있었고, 다른 쪽의 그는 같은 검으로 문을 열고 있었다. 얼굴은 오래 닳아 알아볼 수 없었지만, 손에 쥔 검의 균열만은 방랑자의 것과 같았다.

황혼의 문지기는 방랑자에게 죄를 묻지 않았다. 대신 기억하지 못한 채 짊어진 책임을 물었다. 봉인은 누군가 닫았기에 남아 있었고, 누군가 열었기에 지금 흔들리고 있었다.

둘 중 하나만 진실이어야 한다고 방랑자가 말했지만, 성소는 아무 대답도 하지 않았다. 문은 대답 대신 양쪽으로 길게 그림자를 드리웠다.
```

Stage cleared — 닫힌 문과 열린 그림자:

```text
문지기가 사라진 뒤에도 성소의 문은 완전히 닫히지 않았다. 틈새에서는 바람이 아니라 낮은 속삭임이 흘러나왔다. 그것은 방랑자의 이름을 부르려다 끝내 실패하는 소리와 닮아 있었다.

방랑자는 자신이 봉인을 지켰다는 말을 믿을 수 없었다. 하지만 부정할수록 검은 더 선명하게 반응했다. 잃어버린 기억은 그를 변명해 주지 않았다. 오히려 말없이 앞으로 밀어냈다.

그날 성소가 남긴 단서는 하나였다. 밤의 씨앗은 문 안에만 있는 것이 아니다. 문을 닫은 자의 안에도 그 흔적을 남긴다.
```

### Inferno Chasm

Summary:

```text
불길의 균열은 밤의 씨앗을 단순히 태우거나 부술 수 없다는 사실을 드러낸다.
```

Stage unlocked — 타지 않는 씨앗:

```text
잿불 협곡은 끝없이 불타지만, 그 불은 아무것도 끝내지 못한다. 바위는 녹았다가 다시 굳고, 재는 바람에 흩어졌다가도 같은 자리에 내려앉는다. 이곳의 불꽃은 파괴라기보다 반복에 가깝다.

방랑자는 처음으로 밤의 씨앗을 없애야 할 대상으로 생각했다. 이토록 뜨겁게 타오르는 협곡이라면 어둠마저 삼킬 수 있을 것 같았다. 그러나 잿빛 수호자는 그 생각이 행동이 되기 전에 앞을 막아섰다.

씨앗을 태우려 하지 마라. 깨어날 뿐이다.

그 말은 협박이 아니라 경고였다. 밤의 씨앗은 밖에서 흘러든 독이 아니라, 봉인된 기억과 맹세가 뒤엉켜 자라난 핵이다. 불로 태운다고 사라지는 것이 아니라, 그 안에 갇힌 것들이 더 거칠게 깨어날 뿐이었다.
```

Stage cleared — 잿더미 속의 뿌리:

```text
수호자가 무너진 뒤, 협곡의 불길이 잠시 낮아졌다. 그 아래에서 드러난 것은 검은 씨앗이 아니라 금빛 뿌리 하나였다. 뿌리는 땅속으로 내려가지 않고, 방랑자의 검을 향해 뻗어 있었다.

방랑자는 그제야 깨달았다. 자신은 밤의 씨앗을 찾아 헤매는 것이 아니라, 흩어진 봉인의 길을 거꾸로 되짚고 있었다.

태울 수 없는 것은 반드시 파괴해야 할 대상이 아니다. 때로는 비춰 보아야 할 진실이다.
```

### Cursed Tomb

Summary:

```text
저주받은 무덤은 방랑자가 잃어버린 이름이 봉인 안에 남아 있음을 드러낸다.
```

Stage unlocked — 이름 없는 묘비:

```text
저주받은 무덤의 묘비에는 죽은 자의 이름이 새겨지지 않는다. 이곳에는 살아남기 위해 이름을 버린 자들, 약속을 지키기 위해 스스로를 기록에서 지운 자들의 빈 돌만 남아 있다.

방랑자는 가장 깊은 방에서 자신의 검과 같은 문양을 보았다. 그것은 왕의 문장도, 기사단의 표식도 아니었다. 봉인을 닫기 위해 만들어진 열쇠의 표식이었다.

마지막 파수꾼은 방랑자를 침입자로 부르지 않았다. 그는 오래도록 돌아오지 않은 사람을 맞이하듯 말했다. 너는 문을 닫은 마지막 기사였다.

그 순간 무덤의 모든 빈 묘비가 같은 방향으로 갈라졌다. 그 어디에도 새겨지지 않은 이름 하나가, 아직 사라지지 않은 채 봉인 안에서 기다리고 있었다.
```

Stage cleared — 마지막 기사:

```text
마지막 파수꾼은 패배했지만 방랑자를 원망하지 않았다. 그는 방랑자의 적이라기보다, 방랑자가 스스로 잊기로 한 진실을 지키는 문지기에 가까웠다.

방랑자의 이름은 빼앗긴 것이 아니었다. 그는 세계를 닫기 위해 자신의 이름을 봉인 안에 남겨 두었다. 이름을 가진 자는 문을 열 수 있었고, 이름을 버린 자만이 문을 닫을 수 있었기 때문이다.

하지만 봉인은 완전하지 않았다. 밤의 씨앗은 하나의 어둠이 아니라, 잊혀진 맹세들이 씨앗처럼 맺혀 남은 흔적이었다. 하나를 되찾는다고 해서 모든 밤이 끝나는 것은 아니었다.

방랑자는 무덤을 나서며 처음으로 자신의 검을 무겁게 느꼈다. 그것은 무기가 아니라 열쇠였고, 상처가 아니라 길을 가리키는 지도였다.

그리고 어딘가 깊은 밤 아래에서, 아직 깨어나지 않은 또 다른 씨앗이 숨을 쉬고 있었다.
```

Campaign cleared — 남은 밤:

```text
이야기는 여기서 끝나지 않는다. 방랑자가 되찾은 것은 이름 전체가 아니라, 그 이름으로 향하는 첫 음절에 가까웠다.

밤의 씨앗은 아직 하나가 아니다. 세계 어딘가에는 다른 봉인과 다른 맹세, 그리고 다른 이름들이 잠들어 있다. 방랑자가 걸어야 할 길은 끝난 것이 아니라, 이제야 어디로 향해야 하는지 드러났을 뿐이다.
```

## 상세 스토리 확장용 프롬프트

다음에 스토리 본문을 더 다듬거나 영어 번역을 만들 때 사용할 수 있는 프롬프트다.

```text
너는 모바일 2D 오프라인 뱀서라이크 게임 Nightseed Survivor의 판타지 서사 작가다.

목표:
- 인게임 자막은 짧은 요약으로 유지하고, 메인 메뉴 StoryUI에서 읽을 상세 챕터 본문을 작성한다.
- 문체는 어둡지만 과장되지 않은 서정적 판타지다.
- 모바일 UI에서 읽기 쉽게 한 단락은 2~4문장 정도로 짧게 유지한다.

세계관 핵심:
- Nightseed = 밤의 씨앗.
- Nightseed는 악마, 숭배, 계약, 제물 의식이 아니다.
- Nightseed는 세계에 밤을 자라나게 만드는 봉인된 기억의 씨앗이다.
- 방랑자 Vagrant는 자신의 이름과 과거를 잃었다.
- Vagrant의 검은 봉인을 닫았던 열쇠이자 잃어버린 기억의 지도다.
- 보스들은 단순 악당이 아니라 봉인과 기억에 묶인 파수꾼이다.

정보 공개 순서:
1. Forest of Echoes: Nightseed라는 이름이 처음 등장한다. 숲이 방랑자의 이름을 기억하고 있음을 암시한다.
2. Frozen Wastes: Nightseed가 잊혀진 맹세와 이어져 있음을 드러낸다.
3. Twilight Sanctum: 방랑자가 봉인을 지킨 자이자 연 자라는 모순을 보여준다.
4. Inferno Chasm: Nightseed는 단순히 태우거나 파괴할 수 없다는 사실을 밝힌다.
5. Cursed Tomb: 방랑자가 마지막 기사였고, 잃어버린 이름이 봉인 안에 남아 있음을 공개한다.

금지:
- Nightseed를 악마, 사악한 신, 숭배 대상, 계약 대상, 제물 의식과 연결하지 않는다.
- 기존 상용 게임의 문구, 고유명, 화면 구성, 아이콘, 대사를 모방하지 않는다.
- 너무 긴 컷신 대사처럼 쓰지 않는다.

출력 형식:
각 스테이지마다 아래 구조로 작성한다.

Stage:
Summary: 한 문장
Stage unlocked heading:
Stage unlocked body: 2~4개 짧은 단락
Stage cleared heading:
Stage cleared body: 2~4개 짧은 단락

언어:
- 먼저 한국어로 작성한다.
- 필요하면 같은 의미의 영어 버전을 별도로 작성한다.
```

## 완료 기준

- 인게임에서는 기존처럼 짧은 진행 자막만 표시된다.
- StoryUI에서는 별도 상세 챕터 본문을 읽을 수 있다.
- 해금되지 않은 스테이지는 상세 스포일러를 보여주지 않는다.
- 클리어 후 추가 단서가 필요한 경우 클리어 조건에 맞춰 표시된다.
- `story_dialogues.json`와 `story_chapters.json`의 역할이 문서에 명확히 구분된다.
- Godot headless 실행 또는 실행 불가 사유가 기록된다.
