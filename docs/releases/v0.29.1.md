## 🌙 v0.29.1 — PGS·AdMob 플러그인 ClassNotFoundException 수정

### 🎯 핵심 변경

사용자 보고 "여전히 순위표가 동작 안 한다"를 폰 logcat 으로 직접 진단한 결과, **v0.24.0 minifyEnabled 활성화 이후 모든 빌드**에서 PGS 와 AdMob 플러그인이 silently 안 동작하고 있던 진짜 원인을 찾아 수정했습니다.

---

### 🐛 근본 원인

`GodotPluginRegistry.loadPlugins` 가 AndroidManifest 의 meta-data 를 읽어 reflection 으로 플러그인 클래스를 instantiate 하는데, 두 클래스 모두 ClassNotFoundException:

```
W/GodotPluginRegistry: Unable to load Godot plugin GodotPlayGameServices
  java.lang.ClassNotFoundException:
  com.jacobibanez.plugin.android.godotplaygameservices.GodotAndroidPlugin

W/GodotPluginRegistry: Unable to load Godot plugin PoingGodotAdMob
  java.lang.ClassNotFoundException:
  com.poingstudios.godot.admob.ads.PoingGodotAdMob
```

`proguard-rules.pro` 가 `com.godot.plugin.**` / `org.godotengine.plugin.**` 만 keep 했지만 실제 플러그인 코드는 **서드파티 namespace**(`com.jacobibanez.**`, `com.poingstudios.**`)에 있어 R8 이 stripping. 그 결과 `LeaderboardManager.is_supported()` 가 항상 false → 모든 PGS/광고 호출이 no-op.

---

### 🔧 수정

[proguard-rules.pro](../../godot/android/build/proguard-rules.pro) 에 두 namespace `-keep` 룰 추가:

```
-keep class com.jacobibanez.plugin.android.godotplaygameservices.** { *; }
-dontwarn com.jacobibanez.plugin.android.godotplaygameservices.**
-keep class com.poingstudios.godot.admob.** { *; }
-dontwarn com.poingstudios.godot.admob.**
```

mapping.txt 로 두 클래스 모두 원래 이름 그대로 보존 확인 (난독화 X).

---

### 📌 검증 주의사항

로컬 install 한 APK 는 **upload cert** (`37:55:...`) 로 서명되고 PGS 는 **Play App Signing cert** (`DA:65:...`) 만 인증합니다. 그래서 로컬에서는 플러그인 로드는 성공하지만 sign-in 은 여전히 실패하는 게 정상. 완전한 검증은 Play Store internal testing 트랙에 이 AAB 를 올린 뒤 진행해야 합니다.
