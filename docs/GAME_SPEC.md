# GAME_SPEC

# Project: Nightseed Survivor

## 1. 프로젝트 개요

### 게임 이름

Nightseed Survivor

### 프로젝트 코드명

NIGHTSEED

### 장르

2D 뱀서라이크 / 오토 배틀 생존 액션 / 로그라이트

### 한 줄 설명

플레이어는 어둠에 잠식된 숲에서 자동으로 발동되는 무기와 성장 선택지를 활용해 몰려오는 적들을 10분 동안 버티는 2D 생존 액션 게임이다.

### 핵심 목표

- 이동은 단순하게
- 공격은 자동으로
- 성장은 선택 기반으로
- 한 판은 짧고 반복 가능하게
- MVP는 작게 만들고, 이후 무기/캐릭터/스테이지를 확장한다
- 그래픽 애셋이 없는 상태에서도 placeholder로 먼저 핵심 재미를 검증한다

## 2. 추천 기술 스택

### 메인 엔진

Godot 4.x

### 언어

GDScript

### 타겟 플랫폼

1. PC
2. Android
3. 추후 iOS 가능

### 선택 이유

- 2D 게임 개발에 적합하다
- 씬 기반 구조가 바이브 코딩에 유리하다
- 빠른 실행과 테스트가 가능하다
- Android export가 가능하다
- 프로젝트 구조가 가볍다
- 초반 MVP 제작 속도가 빠르다

---

## 3. 게임 핵심 콘셉트

플레이어는 직접 공격하지 않는다.

플레이어는 이동만 조작하고, 무기들은 일정 주기마다 자동으로 발동된다.

적을 처치하면 경험치 보석이 떨어진다.

경험치를 모으면 레벨업하고, 3개의 성장 선택지 중 하나를 고른다.

시간이 지날수록 적의 수와 강도가 증가한다.

10분을 버티면 스테이지 클리어다.

---

## 4. MVP 범위

### MVP 목표

첫 번째 버전에서는 다음을 구현한다.

- 1개 캐릭터
- 1개 스테이지
- 10분 생존 모드
- 자동 공격 무기 5종
- 패시브 업그레이드 5종
- 적 4종
- 보스 1종
- 레벨업 선택 UI
- 경험치 보석
- 골드 획득
- 간단한 영구 강화 시스템
- 로컬 저장
- 게임오버 / 클리어 화면
- placeholder 그래픽 기반 플레이 가능 상태
- 실제 애셋 교체가 쉬운 씬 구조

초기 버전에서는 아래 기능을 만들지 않는다.

- 멀티플레이

- 스테이지 다중 분기

- 상용 게임의 이름, UI, 아이콘, 아트, 문구 복제

---

## 6. 게임 루프

1. 플레이어가 캐릭터를 선택한다.
2. 스테이지에 입장한다.
3. 적들이 사방에서 몰려온다.
4. 플레이어는 이동으로 적을 피한다.
5. 무기는 자동으로 발동된다.
6. 적을 처치하면 경험치 보석과 골드가 떨어진다.
7. 경험치를 모으면 레벨업한다.
8. 레벨업 시 3개의 업그레이드 중 하나를 선택한다.
9. 시간이 지날수록 적 웨이브가 강해진다.
10. 10분 생존 시 클리어.
11. 사망 시 게임오버.
12. 획득한 골드로 영구 강화를 구매한다.
13. 다시 도전한다.

### PC 조작

|---|---|

| 동작 | 방식 |
|---|---|

- 플레이어는 이동에만 집중한다.

## 15. 레벨업 시스템

### 경험치 요구량

초기 공식:

```text
required_xp = 5 + current_level * 4
```

### 레벨업 선택지 규칙

레벨업 시 3개의 선택지를 보여준다.

선택지 후보:

- 아직 최대 레벨이 아닌 보유 무기
- 아직 최대 레벨이 아닌 보유 패시브
- 아직 보유하지 않은 신규 무기
- 아직 보유하지 않은 신규 패시브
- 모든 후보가 최대 레벨이면 회복 또는 골드 선택지 제공

### 선택지 제한

| 항목 | 제한 |
|---|---|
| 최대 무기 수 | 5 |
| 최대 패시브 수 | 5 |
| 무기 최대 레벨 | 5 |
| 패시브 최대 레벨 | 5 |

## 17. 체력 및 피해 시스템

- 적과 플레이어가 접촉하면 피해를 입는다.
- 같은 적에게 연속 피해를 받지 않도록 짧은 무적 시간을 둔다.
- 기본 무적 시간은 0.5초로 한다.
- MVP에서는 레벨업 선택지 또는 드롭으로만 회복한다.

## 18. 영구 강화 시스템

| 강화 | 효과 | 최대 레벨 | base_cost |
|---|---|---:|---:|
| Max HP | 최대 체력 증가 | 10 | 50 |
| Damage | 모든 피해량 증가 | 10 | 80 |
| Move Speed | 이동 속도 증가 | 5 | 100 |
| Magnet | 경험치 획득 반경 증가 | 5 | 70 |
| Gold Bonus | 골드 획득량 증가 | 5 | 120 |

비용 공식:

```text
cost = base_cost * current_level * current_level
```

## 20. 그래픽 애셋 정책

MVP 단계에서는 최종 그래픽 애셋이 없어도 개발을 진행한다.

- 모든 캐릭터, 적, 투사체, 픽업은 단순 도형으로 표현한다.
- 최종 그래픽을 전제로 코드를 작성하지 않는다.
- 로직과 시각 표현을 분리한다.
- 실제 애셋 교체 시 씬 구조를 크게 바꾸지 않도록 한다.
- 애셋 파일 경로는 하드코딩하지 않고 가능한 한 상수 또는 데이터 파일에서 관리한다.

상세 내용은 `docs/ASSET_GUIDE.md`를 따른다.

## 21. MVP 성공 기준

- 처음 실행한 사람이 조작법을 설명 없이 이해할 수 있다.
- 1판을 10분 안에 끝낼 수 있다.
- 30초 안에 첫 레벨업이 발생한다.
- 레벨업 선택이 최소 5번 이상 발생한다.
- 무기 성장 체감이 있다.
- 6분 이후 난이도가 확실히 올라간다.
- 죽어도 골드가 남아서 다시 하고 싶어진다.
- Android 빌드가 가능하다.
- 저장 데이터가 유지된다.
